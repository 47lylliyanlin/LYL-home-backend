# Kiro 记忆系统

基于Ombre-Brain设计的动态记忆系统。

## 📁 目录结构

### permanent/ - 固化记忆
- **用途：** 存储用户的基本信息、重要事实
- **特点：** 永久保留，不会衰减，`importance = 10`
- **例子：** 用户姓名、家人信息、重要喜好

### dynamic/ - 动态记忆
- **用途：** 存储对话中提取的临时信息
- **特点：** 会随时间衰减，使用频率低的会被归档
- **例子：** 最近聊过的话题、临时计划

### feel/ - 自省记忆
- **用途：** AI对对话的感受和反思
- **特点：** 记录情绪体验，帮助AI保持一致的性格
- **例子：** "和用户聊天很开心"、"用户今天心情不好"

### archive/ - 归档记忆
- **用途：** 存放旧的、不常用的记忆
- **特点：** 检索优先级最低，可以定期清理
- **例子：** 3个月前的临时话题

---

## 📝 记忆文件格式

每个记忆文件必须包含YAML frontmatter：

```markdown
---
id: mem_permanent_001              # 唯一ID
importance: 10                     # 重要度 1-10
valence: 0.7                       # 情感效价 0-1（负面→正面）
arousal: 0.3                       # 情感唤醒度 0-1（平静→激动）
tags: [用户, 基本信息]             # 标签列表
created: 2026-06-24T22:00:00      # 创建时间
last_used: 2026-06-24T22:00:00    # 最后使用时间
use_count: 0                       # 使用次数
type: permanent                    # 记忆类型
---

记忆内容写在这里。
可以是多行文本。
```

## 🔍 记忆检索规则

### 1. 向量语义搜索
使用embedding找到语义相关的记忆

### 2. 衰减分计算
```python
decay_factor = 1 / (1 + days_since_last_used * 0.05)
score = importance * decay_factor * (1 + use_count * 0.1)
```

### 3. 分类优先级
permanent > dynamic > feel > archive

## 🔄 记忆生命周期

### 创建
1. 对话后，Claude判断是否有需要记住的信息
2. 提取关键信息并评估重要度
3. 保存到 `dynamic/` 目录

### 使用
1. 检索时读取记忆
2. 更新 `last_used` 时间
3. `use_count` +1
4. 常用的记忆 `importance` 会增加

### 衰减
1. 长时间不使用的记忆分数降低
2. 低分记忆（score < 2）自动移到 `archive/`

### 固化
1. 非常重要的记忆（importance >= 9）
2. 使用次数超过10次
3. 手动标记为permanent
→ 移到 `permanent/` 目录，设置 `importance = 10`

## 🎯 情感坐标说明

### valence（效价）- 情绪的正负
- 0.0-0.3: 负面（悲伤、生气、失望）
- 0.4-0.6: 中性（平常对话）
- 0.7-1.0: 正面（开心、兴奋、感激）

### arousal（唤醒度）- 情绪的强度
- 0.0-0.3: 平静（安静、放松、无聊）
- 0.4-0.6: 中等（日常对话）
- 0.7-1.0: 激动（兴奋、焦虑、愤怒）

### 例子

| 记忆内容 | valence | arousal | 说明 |
|---------|---------|---------|------|
| 用户考试考了100分 | 0.9 | 0.8 | 正面+激动（开心） |
| 用户的猫生病了 | 0.2 | 0.6 | 负面+焦虑（担心） |
| 今天天气不错 | 0.6 | 0.3 | 正面+平静 |
| 用户喜欢吃火锅 | 0.7 | 0.4 | 正面+中等（喜好） |

## 🛠 维护建议

### 每周
- 检查 `dynamic/` 中的低分记忆，移到 `archive/`
- 检查高频使用的记忆，考虑固化

### 每月
- 清理 `archive/` 中的超旧记忆（>3个月）
- 更新 `permanent/` 中过时的信息

### 手动操作
- 用户要求删除某些记忆
- 更正错误信息
- 合并重复记忆
